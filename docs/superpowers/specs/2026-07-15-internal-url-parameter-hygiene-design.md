# 内部 URL 校验参数清理设计

## 背景与目标

部署验收曾使用 `?verify=<Git 提交短哈希>` 绕过缓存和识别线上版本。例如 `?verify=129535d` 中的值恰好对应当时发布的 Git 提交。它不是用户编号、检测编号或检测日志，页面也不读取它；但它长期留在地址栏会让用户误以为网站正在跟踪或上报本次检测。

本次修改需要同时达成以下目标：

- 在任何统计配置执行前，从主页和 404 页地址栏移除内部部署校验参数。
- 不刷新页面，不重新运行检测，不新增网络请求，也不改变浏览器前进/后退历史。
- 完整保留业务参数、推广参数、参数顺序、重复参数、原始编码及 URL 锚点。
- 部署验收不再依赖用户可见的查询参数。

## 方案比较

### 方案一：只停止生成新链接

以后不再使用 `?verify=`，但不处理已经打开或被转发的旧链接。

优点是没有运行时代码；缺点是旧链接仍会保留误导性参数，而且参数仍可能出现在统计记录中。该方案不能完整解决问题。

### 方案二：使用 `URLSearchParams` 删除

在页面启动时调用 `URL.searchParams.delete()`，再通过 `history.replaceState()` 写回地址栏。

实现直观，但 `URLSearchParams` 会重新序列化所有剩余参数，可能改变空格、转义字符等编码形式。参数语义通常不变，但不能满足“非内部参数原样保留”的要求。

### 方案三：在页面头部按原始查询片段过滤

在主页和 404 页 `<head>` 的最早位置运行同步、内联的清理函数。函数只解析参数名用于白名单匹配，未命中的查询片段保持原始字节、顺序和重复次数不变。

这是采用的方案。它能在 GA 初始化和外部资源加载前完成清理，同时把对正常 URL 的影响限制在明确的内部参数上。

## 清理规则

内部参数采用大小写敏感的精确白名单：

- `verify`
- `deploy`
- `check`
- `qa`

规则如下：

1. 删除上述参数的所有实例，包括无值参数和重复参数。
2. 参数名先按 URL 查询规则解码再匹配，因此等价的百分号编码名称也会被清理。
3. 其他查询片段保持原始顺序、值、编码和重复次数。
4. 路径与 `#锚点` 原样保留。
5. 全部内部参数删除后若没有普通参数，不保留空的 `?`。
6. 使用 `history.replaceState(history.state, "", cleanUrl)`，不新增历史记录、不触发页面刷新。
7. URL 解码或 `replaceState` 在特殊环境中失败时捕获异常并保持原页面可用；清理失败不得阻断检测或 404 页面。

## 执行位置与数据流

主页的清理脚本放在 `index.html` 的基础 meta 标签之后、样式表、应用脚本和 `dataLayer/gtag` 初始化之前。

404 页不会加载 `app.js`，因此在 `404.html` 中放置同样的同步清理逻辑，并确保它位于现有 GA 初始化之前。404 页目前会在解析阶段立即把 `gtag("config")` 放入队列，仅外部统计脚本延迟加载；因此不能依赖延迟任务或主页脚本补救。

执行顺序为：

1. 浏览器解析页面头部。
2. 同步过滤内部 URL 参数并替换当前历史条目。
3. 初始化 `dataLayer/gtag`。
4. 页面正常启动检测。
5. 延迟加载 GA；GA 读取到的页面地址已经不包含内部参数。

## 部署验收

发布后不再通过 `?verify=<commit>` 验证版本，改用以下无用户追踪歧义的流程：

1. 核对 GitHub Pages 构建提交与远端 `main` 提交一致。
2. 分别下载本地文件、GitHub Raw 文件与 GitHub Pages 文件并比对 SHA-256。
3. 浏览器冒烟测试使用干净的正式地址，并在验收环境中禁用缓存。

这样可以验证部署版本，同时不触发页面统计，也不会把提交号放入用户可见 URL。

## 回归测试

在现有确定性 Playwright 套件中增加两个独立场景。

### 主页场景

访问包含四个内部参数、两个普通参数和锚点的主页 URL，并断言：

- 四个内部参数全部从当前 URL 删除。
- 普通参数、原始顺序、重复项、编码和锚点保留。
- `gtag("js")` 与 `gtag("config")` 入队时记录的 URL 均已清理。
- 综合分能够正常生成，证明 URL 清理不阻断自动检测。

### 404 场景

直接访问带内部参数、普通参数和锚点的 `404.html`，并断言：

- 当前 URL 已清理且普通信息保留。
- 解析阶段入队的 `gtag("config")` 看到的是清理后的 URL。
- 页面标题和返回首页入口仍正常显示。

统计顺序测试通过预先包装 `window.dataLayer.push` 记录每次入队时的 `location.href`，不连接真实 GA，也不依赖网络时序。

## 非目标

- 不把 `verify` 解释为用户身份或检测结果。
- 不清理未列入白名单的任意查询参数。
- 不改变 GA 的现有延迟加载策略或隐私披露文案。
- 不在本次修改中重构 GitHub Pages 发布架构或引入新的 CI 工作流。
