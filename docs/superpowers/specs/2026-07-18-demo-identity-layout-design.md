# AiSignalGuard 身份分析 Demo 设计规格

日期：2026-07-18
状态：已确认（落实 2026-07-18 最终版审查建议）
演示入口：`/demo/index-new.html`

## 目标

在不改变根目录 `index.html` 当前线上体验的前提下，制作一个可完整运行的身份分析演示版。演示版继续使用现有 IP、ASN、ISP、DNS、WebRTC、浏览器、语言、时区、AI 服务与网络检测能力，只重构信息层级、结果叙事、导航和移动端交互。

最终体验聚焦四个问题：

1. 当前选择的目标画像是什么。
2. 环境与目标画像的匹配程度如何。
3. 哪些信号支持匹配，哪些信号拉低匹配度。
4. 用户可以优先核对或调整什么。

## 设计判断

- 模式：Redesign - Preserve。保留品牌、检测能力、评分口径和已有数据源。
- 视觉基调：克制、可信、轻科技；暖白底色、低饱和绿色主强调、琥珀色提示、红色差异。
- 设计参数：`DESIGN_VARIANCE 4`、`MOTION_INTENSITY 3`、`VISUAL_DENSITY 6`。
- 主题：沿用当前统一浅色主题，避免演示版与主站在同一次浏览中发生视觉断裂。
- 动效：只保留选择、按钮、折叠和复制状态反馈；不增加装饰性滚动动画。
- 图标：画像和加载状态统一使用 Tabler Icons 的 MIT 授权 SVG 图标，不再使用系统 Emoji。

## 文件架构

演示版采用隔离副本，避免根页面行为被条件分支影响：

- `demo/index-new.html`：从根 `index.html` 复制，调整资源路径、页面结构、文案与工具栏。
- `demo/styles-new.css`：从根 `styles.css` 复制，专门承载演示版布局和响应式规则。
- `demo/app-new.js`：从根 `app.js` 复制，保留检测逻辑，重构身份结果渲染和演示版交互。
- `identityProfiles.js`、`identityAnalysis.js`：继续共享，确保评分模型只有一个事实来源。
- `build/sites-vite-plugin.js`：将 `demo/` 纳入静态构建产物，不改变根页面路由。
- `tests/e2e-smoke.mjs`：增加 demo 专属场景，同时保留根页面现有回归断言。

隔离副本的代价是演示脚本和样式会暂时重复。这里优先保证根线上页面零行为变化；待演示版确认后，再单独规划共享 UI 层的合并，不在本次上线中冒险改造 7000 行检测脚本。

## 首页身份选择

### 内容

保持三个可见画像，顺序不变：

1. AI 用户
2. 自媒体创作者
3. 跨境商家

美国普通用户继续隐藏。自媒体创作者和跨境商家增加“目标市场：美国”信息，避免用户忽略美国地区约束。

### 图标

- AI 用户：Tabler `sparkles` 或 `cpu`。
- 自媒体创作者：Tabler `video`。
- 跨境商家：Tabler `shopping-bag`。
- 通用分析与加载：Tabler `world`。

图标使用与主视觉一致的浅绿背景和深绿描边，不依赖操作系统 Emoji 字体。

### 倒计时

- 无选择时按钮文案为“使用通用画像开始分析 (6s)”并保持可点击状态。
- 选择画像后按钮改为“开始分析：画像名称”，自动倒计时取消。
- 点击、触摸、滚动、键盘聚焦等探索行为会暂停自动进入，避免移动端还未看完卡片就跳转。
- 暂停后按钮仍可手动使用通用画像开始分析，不出现不可用灰态。
- 辅助技术状态文案明确说明倒计时与暂停原因。

## 分析进行中

- 标题包含当前画像，例如“正在分析 AI 用户环境”。
- 使用统一 SVG 画像图标。
- 进度条下方展示具体阶段：浏览器环境、出口网络、DNS 与 WebRTC、服务路径、综合评分。
- 保留现有确定性进度算法，不伪造新的完成时间。

## 结果页信息架构

结果页按以下顺序呈现：

1. 身份总结
2. 支持匹配与拉低匹配度的信号
3. 优先核对建议
4. 环境信号概览
5. 完整评分依据（默认折叠）
6. 高级网络诊断（默认折叠）

### 身份总结

- 选定画像显示“目标匹配度”。
- 通用画像显示“环境一致性分”。
- 保留分数、证据覆盖率、目标画像和一句话总结。
- 结果文案只描述环境相似度，不判断真实身份。
- 总结卡压缩纵向占用，分数与说明形成清晰的左右层级；手机端上下排列。

### 匹配原因

- 左侧：支持匹配的信号。
- 右侧：拉低匹配度的信号。
- “尚未确认”作为次要折叠内容，只有存在未确认信号时才显示。
- 每类最多优先展示三项，其余证据放入完整评分依据，避免与信号卡重复。

### 建议

- 标题改为“优先核对建议”。
- 只显示由明确差异或部分匹配信号生成的建议。
- 无需调整时使用简短中性文案，不展示大面积空状态卡。

### 环境信号概览

- 两列卡片，手机端单列。
- 卡片只承担“信号名称、状态、核心证据”三个信息，不重复解释评分规则。
- 证据正文桌面和手机均不小于 13.5px。

### 完整评分依据

- 使用原生 `details/summary`，默认关闭，支持键盘和辅助技术。
- 桌面表格列宽按“信号 18%、状态 12%、权重 10%、贡献 12%、证据 48%”分配。
- 手机端转换为堆叠评分条目，不允许横向滚动。

### 高级网络诊断

- 将旧网络风险分、出口 IP、身份信号、泄漏、连通、多源、AI 路径、AI 状态、指纹和路由追踪整体放入一个默认关闭的折叠区。
- 标题为“高级网络诊断（独立参考，不计入身份匹配分）”。
- 展开后保持全部既有检测与交互能力。
- 网络信号参考分不再与身份匹配分同时出现在首屏。

## 导航

结果页一级导航精简为五项：

1. 身份总结
2. 匹配原因
3. 调整建议
4. 环境信号
5. 高级诊断

导航保持单行；手机端允许水平滚动并显示可滚动提示。内部高级诊断区不再将每个旧模块提升为一级导航。

## 浮动工具栏

演示版仅保留五项，顺序固定：

1. 重新测试
2. 分享给 AI（复制结构化报告，使用用户提供的 AI 标志）
3. 分享（复制分享文案）
4. 隐藏 / 取消隐藏
5. 回到顶部

移除 Claude 和 GitHub 浮动入口。GitHub 仍保留在页脚。复制按钮依次呈现“复制中”“已复制”“复制失败”，成功状态停留 2 秒后恢复。隐私按钮只存在“隐藏”和“取消隐藏”两种稳定状态。

## 隐私与可访问性

- 顶部隐私说明改为“分析结果在浏览器生成，联网检测会访问第三方服务”。
- 普通显示不遮盖 IP；开启隐藏后 IPv4 只遮盖最后一段，IPv6 只保留前三组。
- 程序化聚焦的结果标题使用透明或自定义聚焦样式，不显示浏览器原生蓝框；所有可交互控件继续保留清晰的 `:focus-visible`。
- 触控目标不小于 44×44px。
- 页面无横向溢出，折叠状态和复制状态均有 `aria` 反馈。
- 尊重 `prefers-reduced-motion`。

## 验收标准

- `/demo/index-new.html` 可直接加载并完成真实检测流程。
- 根 `index.html` 文件及根页面关键 DOM/工具栏行为不因 demo 发生变化。
- demo 画像卡无 Emoji，目标市场约束可见。
- 无选择倒计时文案正确，交互后自动进入暂停，手动通用分析仍可用。
- 结果顺序符合规格，完整评分与高级诊断默认折叠且可展开。
- 一级导航正好五项，浮动工具栏正好五项。
- 桌面、390px 手机和 300px 窄屏均无横向溢出。
- 全部现有单元、构建和根页面 E2E 回归通过；demo 专属 E2E 通过。
- GitHub Pages 地址返回 200，资源路径无 404。
