# 根页面身份结果收敛设计

## 目标

把根页面结果从“两套评分 + 三套重复解释”收敛为一个清晰的信息路径：

1. 先给出目标身份或通用环境结论。
2. 紧接着解释“为什么像 / 为什么不像”。
3. 立即给出“如何更接近目标身份”。
4. 将身份信号放回对应的真实检测模块，用户可点击展开权重、贡献和证据。
5. 高级网络风险诊断只显示风险状态和风险项数量，不再展示第二个百分制分数。

## 设计判断

- 模式：保留现有品牌、检测能力和 URL 的结构性重构。
- DESIGN_VARIANCE：4。结果页需要稳定、可扫描，不做高变化构图。
- MOTION_INTENSITY：3。只保留 hover、active、展开和复制反馈。
- VISUAL_DENSITY：5。首屏结论简洁，技术证据按需展开。
- 视觉系统：延续现有浅色中性底、绿色主色、圆角 14-18px 的原生 CSS 系统。

## 单一评分原则

“Identity Match Score”和旧“网络信号参考分”并不是同一指标：前者按目标画像动态加权，后者按中国网络风险规则扣分。直接平均或并列展示会制造错误理解。

- 选择具体画像时：只展示一个主要百分制 `Identity Match Score`。
- 使用通用分析时：展示“数字环境综合结论”，不展示主百分制数字。
- 高级网络诊断：展示“发现高风险 / 需留意 / 未发现明确风险 / 检测未完成”和红、黄、未确认项数量；内部旧分数只保留给判定逻辑和回归测试，不出现在可见页面或无障碍树。
- 复制给 AI 的报告：只保留具体画像的主要分数；通用分析写明不预设目标画像；高级网络部分改为风险状态和数量，并区分“未确认风险分区”与不计入风险分区的“其他待确认诊断”。

## 结果信息架构

```text
紧凑身份结论
  ├─ 具体画像：画像名 + Identity Match Score + 一句话
  └─ 通用分析：数字环境综合结论 + 一句话（无数字分）

为什么像 / 为什么不像 / 尚未确认

如何更接近目标身份

真实检测模块
  ├─ 出口 IP：地理位置、网络类型及其权重、贡献、证据
  ├─ 身份信号：时区、语言、浏览器环境及其权重、贡献、证据
  ├─ 网络泄漏：DNS、WebRTC 及其权重、贡献、证据
  ├─ 服务连通性：AI / 创作者 / 商业 / 消费服务及其权重、贡献、证据
  └─ 多源交叉：IP 信誉及其权重、贡献、证据

高级网络风险诊断（默认折叠，无百分制分数）
```

## 身份信号归位

每个身份信号只出现一次，使用原生 `<details>` 嵌入对应真实检测分区。`<summary>` 显示“目标匹配 + 状态”，展开后显示权重、得分贡献和检测证据；不再额外生成重复的完整评分表。展开状态在无关的页面刷新渲染中保持，新一轮身份分析时统一复位；支持鼠标、触屏和键盘。

| 身份信号 | 对应检测分区 |
| --- | --- |
| location、network | `sec-ip` |
| timezone、language | `sec-identity` |
| dns、webrtc | `sec-leak` |
| services、consumer_services、creator_services、ads_environment、commerce_services、ai_services | `sec-conn` |
| reputation | `sec-multi` |
| browser | `sec-fp` |

## 高级网络风险诊断

- 容器改为默认关闭的 `<details id="sec-score">`。
- 摘要区显示总状态、逐条风险信号数量、需留意信号数量和未确认分区数量；风险数量与下方可定位的风险项一一对应。
- 展开后保留六个风险节点、逐项提示、判定口径、规则与隐私说明。
- 原 `#score-number` 和 `#score-ring` 作为 `aria-hidden` 的兼容钩子存在，不可见且不进入无障碍树，避免破坏现有判定回归。

## 浮动工具栏

只保留五项，顺序固定：

1. 重新测试
2. 复制给AI诊断
3. 分享
4. 隐藏 / 取消隐藏
5. 回到顶部

移除 ChatGPT、Claude 和 GitHub 浮动入口；GitHub 仍保留在页脚。删除 GitHub Star API 请求。复制按钮的可见文案、`aria-label` 和 `title` 在复制中、已复制、失败与 2 秒恢复后始终同步。

## 响应式与无障碍

- 结果标题、原因、建议在 320px 宽度下单列显示，不产生页面级横向滚动。
- 五个浮动按钮在移动端保持一行，触控区域不小于 40px。
- 所有展开控件使用原生 `<details>/<summary>`；状态文本不只依赖颜色，匹配、部分匹配、不匹配和待确认的小字号状态均满足 4.5:1 对比度。
- 具体画像结果聚焦标题；通用结果使用同一标题焦点机制。
- 复制状态通过可见文本、`title`、`aria-label` 和 live region 同步反馈。

## 文件影响范围

- `index.html`：结果区顺序、高级诊断容器、五项工具栏、资源缓存版本。
- `app.js`：身份结果模板、信号归位、风险摘要、复制状态、报告、导航顺序、删除 Star 请求。
- `styles.css`：紧凑结论、前置原因和建议、嵌入式信号、折叠诊断、状态对比度、五按钮响应式布局。
- `tests/e2e-smoke.mjs`：根页面信息架构、单一评分、信号归位、工具栏状态、报告和移动端回归。
- `demo/index-new.html`、`demo/app-new.js`：统一“复制给AI诊断”文案并删除无用 Star 请求。
- `README.md`：更新结果层级和工具栏说明。
- 构建产物：`app.min.js`、`styles.min.css`、`demo/app-new.min.js`、`demo/styles-new.min.css`。
