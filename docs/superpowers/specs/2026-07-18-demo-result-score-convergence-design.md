# Demo 结果评分与信息收敛设计

## 背景

Demo 结果页同时展示“环境一致性分”和“网络信号参考分”。前者由目标画像的动态权重计算，后者是面向中国暴露风险的扣分制诊断。两套分数都使用 `/100` 的主视觉，用户会自然理解为同一指标，从而把正常差异视为计算错误。

同时，结果页依次展示原因、建议、环境信号和完整评分表，同一份证据被重复呈现，主结论被大量技术文本淹没。根页面已经验证了更清晰的结果架构，可以作为 Demo 的收敛基线。

## 设计决策

1. 只允许一个面向用户的主评分。
   - 选择 AI 用户、自媒体创作者或跨境商家，且核心证据达到评分就绪条件时，展示唯一的“目标匹配度 /100”。
   - 选定画像但核心证据不足时，使用“证据不足 / 暂不评分”的明确终态，不显示 `/100`，也不继续显示“分析中”。
   - 跳过选择时不预设目标画像，因此不显示数值分，只展示“数字环境综合结论”和证据覆盖率。
   - 高级网络诊断不再显示数值分，改为风险结论及“高风险、需留意、未确认”三类计数。

2. 结果首屏只保留三层信息。
   - 身份总结：一句话结论、唯一评分（如适用）、覆盖率、重新选择入口。
   - 为什么像、为什么不像：每侧最多三条最重要信号。
   - 如何更接近目标身份：最多三条可执行建议。

3. 删除重复的独立“环境信号概览”和“完整评分依据”表格。
   - 不删除任何检测或评分能力。
   - 将画像信号按语义嵌入出口 IP、身份信号、网络泄漏、网络连通、多源交叉和浏览器指纹模块。
   - 每项画像信号使用默认闭合的 `<details>`，展开后显示权重、得分贡献和完整证据。

4. 高级网络诊断使用渐进式披露。
   - 折叠摘要直接给出风险结论和三类计数。
   - 展开后中心只显示“风险信号 / 点击节点查看”，六个节点保留原有状态、定位和工具提示。
   - 原网络分仍保留为内部兼容数据，不在可见界面或复制报告中作为第二评分出现。

5. AI 诊断报告与页面语义一致。
   - 通用分析不输出“环境一致性分”。
   - 选定画像只输出一次“目标匹配度”。
   - 高级网络诊断输出风险结论及计数，不输出“网络信号参考分”。

6. 根页面网络连通面板修复。
   - `.conn-panel` 本身取消内边距。
   - 新增 `.conn-panel-body` 承担原有内容内边距。
   - 动态插入的身份匹配模块因此紧贴面板边界，并由面板圆角完整裁切；原连通内容间距保持不变。

## 视觉与交互基线

- 模式：保留式重设计。
- 视觉语言：沿用现有浅色、可信、低噪声风格。
- DESIGN_VARIANCE：4。
- MOTION_INTENSITY：2，仅保留 hover、active、details 展开等必要反馈。
- VISUAL_DENSITY：5，结论优先，技术证据按需展开。
- 桌面和移动端均不得出现横向滚动；交互目标不小于 44px；键盘焦点保持可见。

## 验收标准

- Demo 通用结果没有任何可见 `/100` 分数。
- Demo 选定画像且核心证据就绪时，结果恰好有一个可见 `/100` 分数。
- Demo 选定画像但核心证据不足时，显示“证据不足 / 暂不评分”，不出现 `/100`、“分析中”或“证据收集中”。
- 高级诊断折叠和展开时都不出现第二个可见数字分。
- 结果根区只有总结、原因、建议三块，不存在独立评分表。
- 每个画像信号只在对应真实检测模块出现一次，默认闭合，展开后证据完整。
- 复制给 AI 的报告不包含“网络信号参考分：N/100”。
- 证据不足时，复制给 AI 的报告写明“目标匹配度：证据不足，暂不评分”，不输出占位分数。
- 根页面 `.identity-section-match` 在桌面和移动端均与 `.conn-panel` 内边界贴齐。
- 全量单元、站点和 E2E 回归通过，源文件与 min 文件同步。
