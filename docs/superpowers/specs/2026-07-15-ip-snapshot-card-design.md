# 出口 IP 截图名片设计

## 决策

采用已确认的 A「截图名片型」。原稿位于 `.superpowers/brainstorm/23508-1784109015/content/ip-card-layouts.html`，生产实现保持其信息层级、桌面四列和手机两列布局。

名片新增在“出口 IP”面板顶部，现有“出口 IP 质量”和“一致性核对”两行原样保留在下方。名片只承担快速阅读和一屏截图，不重复多源表格，也不增加百分比、信誉分或无法由免费数据源验证的结论。

## 信息层级

顶部区域包含：

- 标签“当前公网出口”。
- 主地址：优先 IPv4；没有 IPv4 时使用 IPv6。
- 双栈副地址：主地址为 IPv4 且检测到 IPv6 时，显示 `IPv6 · 地址`。
- 网络类型胶囊：显示规范化后的“住宅宽带、移动运营商、运营商网络、机房 / 云网络、VPN / 代理、类型待确认”等文本；颜色沿用出口 IP 最终状态。

事实栏固定为四项：

1. 位置
2. ASN
3. 组织 / ISP
4. 服务商类型

所有字段从 `state.exitIps[]` 的聚合结果直接读取，不解析现有行文案。名片使用主地址对应的同一条聚合结果，避免把 IPv4 的地址与 IPv6 的 ASN、组织混成一条不存在的路径。

## 证据与文案口径

- 位置只有在 `geoOk === true` 且没有 `geoConflict` 时才显示；否则显示“地区未确认”或“地区证据分歧”。
- ASN 和组织只描述“来源返回的字段”，不写“多源确认”。
- 服务商类型的原始值来自显式 `type` 字段。
- `Residential / Consumer / Fixed line / Broadband` 归一为“住宅宽带”。
- `Mobile / Cellular` 归一为“移动运营商”。
- `ISP / Telecom / Cable / Fiber` 归一为“运营商网络”。
- `Hosting / Datacenter / Cloud / Server / Colo / VPS` 归一为“机房 / 云网络”。
- `VPN / Proxy / Tor / Anonymous` 归一为“VPN / 代理”。
- 同时存在住宅与机房、VPN、代理等冲突类型时显示“类型证据分歧”。
- 没有显式类型、但组织名称命中机房启发式时，只显示“疑似机房 / 云网络”，不得扩写成已确认 VPN 或代理。
- 没有足够类型证据时显示“类型待确认”，不因组织名称看似运营商就宣称住宅或原生 IP。

## 生命周期

名片状态以 `state.ipDiscoveryDone` 为第一门槛：

| 数据状态 | `data-state` | 视觉状态 | 文案 |
|---|---|---|---|
| 尚无地址，检测未完成 | `pending` | 灰色 | 正在读取出口 IP |
| 已有临时地址，检测未完成 | `provisional` | 灰色 | 地址可显示，胶囊为“情报核对中” |
| 检测完成且有地址 | `ready` | 复用出口行的绿 / 黄 / 红 | 显示最终字段与网络类型 |
| 检测完成但没有地址 | `unavailable` | 灰色 | 无法读取出口 IP，字段均为未确认 |

临时结果一律不得提前染绿、染黄或染红。失败态不得残留上一轮 IP、ASN、组织或类型。

## 隐私与安全

- IPv4、IPv6、位置、ASN、组织字段添加 `.sensitive`，复用全站隐私模式的模糊规则。
- 网络类型和状态继续可读，保证隐私截图仍能说明判断结果。
- 所有动态内容使用 `escapeHtml()`，包括恶意组织名和来源返回的原始类型。
- 隐私模式只改变显示，不改变 DOM 中的检测事实；“复制给 AI 分析”继续使用真正脱敏的报告管线。

## 响应式

- 桌面：顶部左右分布，事实栏为 `.8fr .7fr 1.35fr 1fr` 四列，列间竖线。
- 手机：主地址字号从 25px 降为 22px；事实栏改为 2×2，第二行用横分隔线；长 IPv6 和长组织名允许任意断行。
- 320px、360px、390px 均不得产生页面级横向滚动。
- 卡片与下面首行之间保留 `#f0f0ed` 分隔线，形成一个完整面板。

## 可访问性

- 使用 `<article>` 表达独立的出口概览，使用 `<dl><dt><dd>` 表达事实字段。
- `data-state`、`data-status` 和可见文字共同表达状态，不能只靠颜色。
- 状态胶囊包含已有 `.dot`，并通过 `aria-label` 说明当前状态。

## 验证标准

- 名片是 `#sec-ip .panel` 的第一个子项。
- 后续两行仍严格为 `ip`、`consistency`。
- 完整 fixture 正确显示 IP、位置、ASN、组织和显式类型。
- 双栈 fixture 同时显示 IPv4 与 IPv6。
- 住宅显式标签优先于组织名称启发式，不误报机房。
- 地区缺失、国家冲突、重测失败和检测中状态不被误标可信。
- 隐私模式对敏感字段生效。
- 手机长 IPv6、长组织名和右下工具栏同时存在时无页面级溢出。
