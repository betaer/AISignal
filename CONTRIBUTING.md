# 贡献指南

感谢你愿意帮助改进 AI Signal Guard。这个项目的重点是浏览器端网络诊断、隐私自查和安全研究。

## 适合提交的内容

- 新的 IP 情报源、AI 路径目标或服务状态源。
- DNS / WebRTC / 指纹检测的误判样本和修正。
- 不同网络环境、系统、浏览器下的兼容性问题。
- README、FAQ、发布材料和截图改进。
- 无障碍、移动端布局和隐私模式体验改进。

## 提 issue 前请尽量提供

- 浏览器和系统版本。
- 是否开启代理、TUN、DoH、IPv6。
- 问题发生在哪个检测模块。
- 开启隐私模式后的截图，或使用“复制诊断摘要”生成的文本。
- 期望结果和实际结果。

请不要提交真实账号、完整 IP、DNS 解析器、组织信息或浏览器指纹值。需要讨论时优先打码。

## 本地运行

```bash
python3 -m http.server 8080
```

访问 `http://localhost:8080/`。

## 文案边界

项目文案应使用“网络诊断、隐私自查、安全研究、信号一致性检查、访问异常排障”。不要使用“绕过风控、防封、保证账号可用”等表述。
